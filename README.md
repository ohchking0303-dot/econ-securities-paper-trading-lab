# Econ Securities Paper Trading Lab

A paper-only trading and mock investment research lab built for coursework and self-directed study in Principles of Economics and Introduction to Securities Investment.

This project is designed to help students study market behavior, portfolio decisions, trading strategies, transaction costs, volatility, drawdown, and risk-adjusted performance through reproducible simulations.

The project does not place live orders, does not provide financial advice, and does not require private exchange API keys by default.

## Academic Context

I am currently taking Principles of Economics and Introduction to Securities Investment. As part of my coursework and related mock investment practice, I built this open-source project to make trading experiments more structured, reproducible, and risk-aware.

Instead of relying only on manual mock trading or spreadsheet-based tracking, this repository provides a Python-based framework for:

- collecting public market data,
- testing investment hypotheses,
- running paper-only trading simulations,
- comparing strategy performance,
- analyzing drawdown and volatility,
- measuring transaction-cost effects,
- generating reports for review.

## Purpose

The purpose of this project is educational.

It helps students and beginner researchers understand how investment decisions perform under different market conditions. The focus is not on predicting profits, but on learning how risk, cost, leverage, volatility, and market regimes affect simulated investment outcomes.

## Key Features

- Paper-only trading simulation
- Public market-data analysis
- Mock investment experiment tracking
- Backtesting and walk-forward testing
- Portfolio and position-size experiments
- Risk metrics such as MDD, Sharpe ratio, profit factor, volatility, and cost drag
- Report generation for coursework review
- Safe defaults with no live trading and no private account access

## Safety Policy

This project is intentionally paper-only.

It does not include:

- live order execution,
- real-money trading automation,
- financial advice,
- profit guarantees,
- private account access,
- API-key storage by default.

All results are simulated and should be used only for educational and research purposes.

## Example Use Cases

- Comparing passive and active investment strategies
- Studying transaction costs in short-term trading
- Measuring drawdown during volatile markets
- Testing portfolio allocation rules
- Studying leverage sensitivity in a controlled simulation
- Creating reports for mock investment assignments
- Understanding the gap between backtest results and paper-trading observations
